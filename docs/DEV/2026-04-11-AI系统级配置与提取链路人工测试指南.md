# AI 系统级配置与提取链路人工测试指南（2026-04-11）

> 适用范围：验证码提取策略与 AI 系统级配置迁移专项（Web + External + Temp Mail）

---

## 1. 启动与环境前置

1. 推荐启动方式：
   - `python start.py`（优先）
   - 或 `python web_outlook_app.py`（现已支持自动加载 `.env`）
2. 校验健康接口：`GET /healthz` 返回 200。
3. 使用管理账号登录系统。

### 1.1 关键前置检查

1. `SECRET_KEY` 必须与当前数据库加密历史一致；若变更会导致 `enc:` 字段解密失败。
2. `DATABASE_PATH` 必须指向预期数据库文件，避免“连到了空库/新库”。
3. 本地调试建议关闭系统代理对 127.0.0.1 的影响（或显式配置 NO_PROXY）。

## 2. 测试项 A：系统级 AI 配置

路径：`系统设置 -> 基础 -> 验证码 AI 增强`

### A-1 缺项拦截（保存期）

1. 开启 AI 开关。
2. 清空 `Base URL` / `API Key` / `模型 ID` 任一必填项。
3. 点击保存。

预期：

1. 返回错误 `VERIFICATION_AI_CONFIG_INCOMPLETE`。
2. 错误详情包含缺失字段列表。

> 注意：若页面已存在脱敏 API Key 占位（如 `sk-****`）且未修改，该值会被视为“已有配置”，可能导致你体感“缺项没拦截”。

### A-2 完整配置保存

1. 填写完整 `AI 开关 + Base URL + API Key + 模型 ID`。
2. 保存后刷新页面。

预期：

1. 开关状态、Base URL、模型 ID 可回显。
2. API Key 仅脱敏回显，不明文展示。

### A-3 AI 可用性探测（已保存配置，连通性优先）

1. 在 `系统设置 -> 基础 -> 验证码 AI 增强` 点击 `🤖 测试 AI 配置`。
2. 前端会调用：`POST /api/settings/verification-ai-test`。
3. 观察结果区 `verificationAiTestResult` 与接口返回的 `probe`。

预期：

1. 连通性成功（测试目标）：
   - 只要上游返回 HTTP 2xx，即 `ok=true`、`connectivity_ok=true`
2. 契约成功（增强目标）：
   - 若同时满足固定 JSON 契约，则 `contract_ok=true`
3. 契约未通过但连通正常：
   - 仍可能 `ok=true`（因为连通性达标）
   - 通过 `contract_ok=false` 与 `probe.error` 观察具体偏差
4. 明确错误类别：
   - `config_incomplete`（配置缺项）
   - `http_error`（上游 HTTP 错误）
   - `request_failed`（请求失败）
   - `invalid_response_format`（响应结构不符）
   - `invalid_ai_output`（AI 输出不符合固定 JSON 契约）

## 6. 常见问题排查

### Q1：提示凭据解密失败（ACCOUNT_CREDENTIAL_DECRYPT_FAILED）

优先检查：

1. 是否使用了错误的 `SECRET_KEY`。
2. 是否连接了错误数据库（`DATABASE_PATH`）。
3. 是否曾重建 .env 导致 SECRET_KEY 变化。

### Q2：感觉缺项拦截没生效

优先检查：

1. 页面是否已有历史配置值（尤其 API Key 脱敏占位）。
2. 是否实际清空并提交了字段。
3. 后端响应是否包含 `VERIFICATION_AI_CONFIG_INCOMPLETE`。

## 10. 解析策略更新（2026-04-11）

用户确认：最终目标以“能提取到验证码/验证链接”为主，不再把 `confidence/schema` 作为硬门槛。

### 10.1 调整点

1. `_parse_verification_ai_content(...)` 放宽：
   - 不再硬校验 `schema_version`。
   - 不再硬校验 `confidence/reason` 类型。
   - 只要 `verification_code` 或 `verification_link` 有一个可用，即接受 AI 输出。
2. `confidence` 归一化：
   - 数值/布尔/字符串统一映射为 `high/low`。

### 10.2 复测结果

1. 切换 `deepseek-ai/DeepSeek-V3.2` 后，`/api/settings/verification-ai-test` 可达到：
   - `ok=true`
   - `connectivity_ok=true`
   - `contract_ok=true`
2. `Qwen/Qwen3-14B` 若仍返回 `choices=null`，依旧无法提取（因无内容可解析）。
