## 1. 功能定义

本专项在“验证码提取”场景落地三项能力：

1. AI 配置系统级闭环（settings）
2. 提取链路加速（规则快路径优先，AI 按需触发）
3. AI 输入/输出固定 JSON 契约

---

## 2. 功能边界

### 2.1 本期包含

1. settings 提供 AI 开关、Base URL、API Key、模型 ID。
2. groups 仅保留规则策略（`verification_code_length` / `verification_code_regex`）。
3. Web / External 提取链路统一读取系统级 AI 配置。
4. AI 输入固定 JSON 模板（固定字段 + 版本字段）。
5. AI 输出固定 JSON 模板（固定字段 + 类型约束）。
6. 非法 AI 输出统一降级到规则结果，且有可观测错误。

### 2.2 本期不包含

1. 多供应商模型路由。
2. 账号级 AI 配置。
3. Prompt 平台化管理。

---

## 3. 系统行为设计

### 3.1 规则与 AI 的执行顺序

1. 优先执行规则提取（快路径）。
2. 当规则结果不足时，才触发 AI 增强。
3. AI 异常不应阻塞返回，应快速回退规则结果。

### 3.2 配置读取顺序

1. 规则读取：request > group > default
2. AI 读取：仅系统级 settings（group 不参与运行期 AI 决策）

### 3.3 AI 配置有效性

当 AI 开关开启时：

1. Base URL 为空 → 配置错误
2. API Key 为空 → 配置错误
3. 模型 ID 为空 → 配置错误

返回明确错误，不静默降级。

---

## 4. 固定 JSON 契约

### 4.1 AI 输入 JSON（固定）

说明：以下为功能契约示例，字段集固定，键名固定。

```json
{
  "schema_version": "verification_ai_v1",
  "task": "extract_verification",
  "mail": {
    "subject": "...",
    "text": "...",
    "html": "..."
  },
  "rules": {
    "code_regex": "...",
    "code_length": "6-6"
  },
  "hints": {
    "code_source": "all"
  }
}
```

### 4.2 AI 输出 JSON（固定）

```json
{
  "schema_version": "verification_ai_v1",
  "verification_code": "",
  "verification_link": "",
  "confidence": "low",
  "reason": ""
}
```

约束：

1. `schema_version` 必须匹配。
2. `verification_code` / `verification_link` 为字符串（可空）。
3. `confidence` 允许值：`high` / `low`。
4. 非法结构视为 AI 输出无效。

说明：V1 采用二档置信度口径（`high/low`），不引入 `medium`。

---

## 5. API 与页面契约

### 5.1 settings API

`GET /api/settings` 与 `PUT /api/settings` 增加/承载：

1. `verification_ai_enabled`
2. `verification_ai_base_url`
3. `verification_ai_api_key`（写入；读取返回 masked）
4. `verification_ai_model`

### 5.2 settings 页面

在系统设置页增加“验证码 AI 增强”配置区：

1. AI 开关
2. Base URL
3. API Key（脱敏展示）
4. 模型 ID

### 5.3 groups 页面

分组弹窗不再展示 AI 字段，仅展示 length/regex。

---

## 6. 错误反馈规范

### 6.1 配置错误

1. AI 开启但配置缺失 → 返回统一配置错误码（具体实现名由后端统一定义）

### 6.2 AI 契约错误

1. AI 返回非 JSON / 非法 JSON / 字段缺失 / 类型错误
2. 系统记录“AI 输出无效”并快速回退规则结果

### 6.3 兼容策略

1. group 侧历史 AI 字段保留兼容读写窗口，但不作为主配置来源

---

## 7. 验收映射

1. settings 可完整维护 AI 配置并安全回显。
2. 提取路径优先规则快路径，AI 按需触发。
3. AI 输入/输出固定 JSON 契约可被验证。
4. 非法 AI 输出不影响主流程返回。
5. Web 与 External 行为一致。
