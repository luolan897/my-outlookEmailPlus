## 1. 技术目标

1. 运行期 AI 配置统一来自 settings。
2. 提取链路实现“规则优先，AI 按需，异常快速回退”。
3. AI 输入/输出采用固定 JSON 契约并可程序化校验。

---

## 2. 分层改造范围

### 2.1 settings 链路

- `outlook_web/repositories/settings.py`
  - 增加 verification_ai_* 读取函数
- `outlook_web/controllers/settings.py`
  - GET/PUT settings 新增 verification_ai_* 字段
  - API key 采用加密存储 + 脱敏回显

### 2.2 前端设置页

- `templates/index.html`
  - 新增“验证码 AI 增强”配置区
- `static/js/main.js`
  - load/save settings 增加 verification_ai_* 字段处理

### 2.3 提取链路

- `outlook_web/controllers/emails.py`
- `outlook_web/services/external_api.py`
- `outlook_web/services/verification_extractor.py`

新增统一行为：

1. 规则快路径先行
2. AI 触发前做系统配置校验
3. AI 输入固定 JSON 构造
4. AI 输出固定 JSON 校验
5. 非法输出回退规则结果

---

## 3. 固定 JSON 契约（技术口径）

### 3.1 输入 JSON 校验

必须字段：

1. `schema_version`
2. `task`
3. `mail.subject`
4. `mail.text`
5. `mail.html`
6. `rules.code_regex`
7. `rules.code_length`
8. `hints.code_source`

### 3.2 输出 JSON 校验

必须字段：

1. `schema_version`
2. `verification_code`（str）
3. `verification_link`（str）
4. `confidence`（enum: high/low）
5. `reason`（str）

输出不合法处理：

1. 记录无效输出事件
2. 不中断主流程
3. 回退规则结果

---

## 4. 兼容策略

1. groups 历史 AI 字段保留迁移窗口，但运行期不作为权威来源。
2. 先“停用运行期依赖”，再做后续 schema 清理。

---

## 5. 错误语义

1. AI 配置缺失：配置错误（400）
2. AI 输出非法：可观测但回退，不直接 500
3. 规则与 AI 均未命中：沿用既有提取未命中语义
