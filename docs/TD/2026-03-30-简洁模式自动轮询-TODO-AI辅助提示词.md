## 一、项目背景

本项目是一个 Outlook 邮箱管理系统，用户可以管理多个 Outlook/IMAP 邮箱账号，简洁模式下以列表形式展示账号摘要信息。新功能目标是：**用户复制邮箱地址后，系统自动轮询该邮箱直到发现新邮件并自动提取验证码**。

### 技术栈
- **后端**：Flask + SQLite + Python unittest
- **前端**：Vanilla JavaScript（无框架）+ Jinja2 模板
- **测试**：Python unittest + Jest + jsdom
- **架构模式**：全局函数命名空间，无模块化

## 三、关键实现约束（必须遵守）

### 3.1 后端约束

1. **无新 API 端点**：复用现有 `/api/settings` GET/PUT 和 `/api/emails/{email}` 接口
2. **使用 INSERT OR IGNORE**：db.py 中添加默认值时用 `INSERT OR IGNORE`，不覆盖已有值
3. **校验范围与前端一致**：
   - `compact_poll_interval`: 3-60 秒
   - `compact_poll_max_duration`: 10-600 秒

### 3.2 前端约束（关键修正点）

这些修正来自外部 AI 审查发现的 **2 个阻塞性问题**：

| 修正点 | 错误做法 | 正确做法 |
|-------|---------|---------|
| DOM 行选择器 | `.compact-account-row` | `.mail-row`（与实际 DOM 一致） |
| 属性选择器转义 | `CSS.escape(email)` | 直接用 `email`（`[onclick*="..."]` 是 HTML 属性匹配） |

**为什么必须修正**：
- `.compact-account-row` 在实际 HTML 中不存在，会导致 `closest()` 返回 `null`
- `CSS.escape()` 会转义 `@` 和 `.` 为 `\\40` 和 `\\2e`，但 HTML 属性值不含这些转义字符

### 3.3 代码结构约束

1. **全局函数模式**：所有轮询引擎函数定义在全局作用域，不用 IIFE 或 ES Module
2. **事件解耦**：`emails.js` 派发 `email-copied` 事件，`mailbox_compact.js` 监听，两文件不直接依赖
3. **双份数据源**：`main.js` 和 `mailbox_compact.js` 各有一份设置变量，通过 `applyCompactPollSettings()` 同步
4. **设置即时生效**：保存后立即调用同步函数，不等待 PUT 响应

### 3.4 UI 关键约束

| 约束 | 值 | 说明 |
|------|---|------|
| 轮询 Toast 时长 | 5000 ms | `COMPACT_POLL_TOAST_DURATION = 5000` |
| showToast duration 参数 | 第 4 参数，默认 `null` | 保持原有默认行为（3000/8000） |
| 按钮最小宽度 | 100px | 不是 120px，减少宽度跳跃 |
| snippet-meta 包含 | from + folder + received_at | 与 `renderCompactAccountList` 一致 |

### 3.5 集成点约束

| 集成点 | 要求 |
|-------|------|
| `renderCompactAccountList()` 末尾 | 必须调用 `reapplyAllCompactPollUI()` |
| 设置加载后 | 必须调用 `applyCompactPollSettings()` |
| 设置保存后 | 必须调用 `applyCompactPollSettings()` |
| baseline 拉取时机 | 先 `compactPollMap.set()` + `updateCompactPollUI()` 再 await baseline |

### 5.3 startCompactAutoPoll 关键顺序

```javascript
async function startCompactAutoPoll(email, accountId) {
    // 1. 如果已在轮询，先停止
    if (compactPollMap.has(email)) {
        stopCompactAutoPoll(email, true);
    }

    // 2. 先创建状态并更新 UI（在 baseline 拉取之前！）
    const state = {
        timer: null,
        baselineIds: new Set(),
        accountId: accountId,
        errorCount: 0,
        startTime: Date.now(),
        isPolling: false,
        uiMissCount: 0
    };
    compactPollMap.set(email, state);
    updateCompactPollUI(email, 'polling', compactPollMaxDuration);
    startGlobalCountdown();

    // 3. 再拉取 baseline
    try {
        const [inboxRes, junkRes] = await Promise.allSettled([...]);
        // ... 处理 baseline
    } catch (e) {
        console.warn('[简洁轮询] 建立 baseline 失败:', e);
    }

    // 4. 立即首次轮询
    pollSingleEmail(email);

    // 5. 设置定时器
    state.timer = setInterval(() => pollSingleEmail(email), compactPollInterval * 1000);
}
```
