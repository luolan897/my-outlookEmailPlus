## 1. 文档目标

本 TDD 只关注“邮箱池项目维度成功复用”的测试设计，不重复展开业务背景与技术方案。

术语约定：

1. **覆盖路径**
   - 指“长期邮箱 + 显式传入 `project_key` + 显式传入 `caller_id` + 显式传入 `task_id`”的测试路径
2. **旧路径**
   - 指“未传 `project_key`”或“不在第一阶段覆盖范围内”的测试路径
3. **第一阶段排除的临时邮箱**
   - 在技术识别上至少包括 `provider='cloudflare_temp_mail'` 或 `account_type='temp_mail'`

本次测试要证明六件事：

1. 长期邮箱在显式传入 `project_key` 时，`success` 后确实直接回到 `available`
2. 同项目只防 success，不防失败 / release / expire
3. `claim-complete` 即使未传 `project_key`，也能依靠 claim 上下文正确写入项目 success 记录
4. 未传 `project_key` 或不在覆盖范围内时，旧行为不被破坏
5. 历史 `used` 长期邮箱迁移后，能重新进入候选池
6. 全局 stats / 外部 API 契约随新语义自然收敛，不引入新的契约漂移

## 3. 测试原则

1. **以自动化为主**：本需求无新增 UI，核心价值在状态机、迁移和接口契约，优先用自动化测试锁定。
2. **先测新语义，再测旧语义兼容**：不能只证明新路径能通，还要证明未传 `project_key` 的旧路径仍是 `success -> used`。
3. **迁移测试必须独立**：历史 `used` 数据迁移不应只靠接口测试间接证明，要有单独迁移断言。
4. **不引入伪业务假设**：测试要体现本轮已确认的迁移口径——历史是否给同项目用过不重要，优先释放长期邮箱资产复用。

## 4. 关键测试矩阵

### 4.1 新语义主路径矩阵

| 场景 | 输入 | 预期 |
|---|---|---|
| N-01 | 长期邮箱 + `project_key` + `success` | `claim-complete` 返回 `pool_status='available'` |
| N-02 | 长期邮箱 + `project_key` + `success` 后同项目再次 claim | 命中同项目 success 记录，返回 `NO_AVAILABLE_ACCOUNT` |
| N-03 | 长期邮箱 + `project_key=A` success 后，`project_key=B` 再次 claim | 可立即再次领取 |
| N-04 | 长期邮箱 + `project_key` + `verification_timeout` | 不写 success 记录，同项目后续仍可重试 |
| N-05 | 长期邮箱 + `project_key` + `claim-release` | 不写 success 记录，同项目后续仍可重试 |
| N-06 | 长期邮箱 + `project_key` + `lease_expired` | 不写 success 记录，同项目后续仍可重试 |

### 4.2 旧语义兼容矩阵

| 场景 | 输入 | 预期 |
|---|---|---|
| O-01 | 长期邮箱 + 未传 `project_key` + `success` | 继续返回 `pool_status='used'` |
| O-02 | `cloudflare_temp_mail` + 传 `project_key` + `success` | 继续走旧语义，不进入“success 后回 available” |
| O-03 | 非覆盖路径 claim-complete | 无需新增 `project_key` 入参 |

### 4.3 迁移矩阵

| 场景 | 初始数据 | 预期 |
|---|---|---|
| M-01 | 历史长期邮箱 `pool_status='used'` | 迁移后回到 `available` |
| M-02 | 历史临时邮箱 / `cloudflare_temp_mail` `pool_status='used'` | 不纳入长期邮箱迁移语义 |
| M-03 | 历史长期邮箱迁回后由原项目再次 claim | 允许再次拿到一次，属于已接受代价 |
| M-04 | 历史 `account_project_usage` 旧 claim 痕迹存在但 success 字段为空 | 不应阻止同项目再次 claim |

### 4.4 契约与统计矩阵

| 场景 | 接口 | 预期 |
|---|---|---|
| C-01 | `/api/external/pool/claim-complete` | 响应结构不变，仅 `pool_status` 值切到 `available` |
| C-02 | `/api/external/pool/stats` | 覆盖路径 success 后，`available` 增加、`used` 不增加 |
| C-03 | 同项目 success 历史命中导致不可分配 | 继续返回通用 `NO_AVAILABLE_ACCOUNT` |
