缺点：需要与源文件保持同步。

**方案 B：fs.readFileSync + eval**
在 setup.js 中读取 `mailbox_compact.js` 文件内容，用正则提取轮询引擎代码段并 eval 到全局作用域。
```javascript

const match = source.match(/\/\/ ={10} 简洁模式自动轮询[\s\S]*$/);
if (match) eval(match[0]);
```
优点：直接测试源文件。缺点：依赖注释标记。

**方案 C（最终推荐）：独立模块**
将轮询引擎代码从 `mailbox_compact.js` 提取到 `static/js/features/compact_poll_engine.js`，在 `mailbox_compact.js` 中通过 `<script>` 标签引入。测试直接 require 该模块。

**TDD 文档采用方案 B 编写用例**（开发时再决定实际策略），用例不依赖具体加载方式。
